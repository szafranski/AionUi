# Aion 远程访问 + 账号体系 PRD v2.1

> 来源：v2.0 原型 CDP 评估发现"登录完还要再开远程"的双步矛盾。本版重新对齐"账号 = 远程"原则，收敛入口和状态机。

---

## 1. 目标

**A. 入口决定行为**：从远程相关入口登录 → 自动激活远程；从纯账号入口登录 → 仅登录账号
**B. 入口收敛**：四个触点（首页 chip / 对话胶囊+地球 / 侧栏账号块 / 设置 toggle）只表达"未开" vs "已开"
**C. 设置页是真相源**：QR / 专属地址 / 设备列表 / 开关都在那里
**D. 强绑定**：退账号 = 远程服务自动停；账号在 = 远程服务可被用户开关

不做：

- 单独的"暂停"概念（用户感知就是 toggle 关 / 开）
- "已登录但远程未开"作为常态曝光（首页 chip 不显示）
- 多账号切换

---

## 2. 用户与场景

**主用户**：在桌面跑长任务的开发者，希望出门后手机继续看进度/追问。

**触发场景**：

- S1 长任务出门：临走/已在外，想手机看进度
- S2 跨设备协作：公司 ↔ 家
- S3 临时设备：朋友电脑/手机浏览器临时用一下

**心智模型**：

> "我的 Aion = 网址 abc12.aion.app，登过我账号的浏览器都能用。"

---

## 3. 状态机

```
                ┌──────────┐
                │  GUEST   │  未登录
                └──────────┘
                  │ 远程入口登录       │ 纯账号入口登录
                  ▼                    ▼
       ┌─────────────────┐    ┌─────────────────┐
       │ ACTIVE 远程已开 │    │ INACTIVE 已登录·关 │
       └─────────────────┘    └─────────────────┘
                  │ 手动 toggle 关 ───→│
                  │←─── 手动 toggle 开 │
                  │
                  │ 网络抖动 → OFFLINE（toggle 仍 ON，状态条变红）
                  │
                  └── 退账号 ──→ GUEST
```

**三态**：GUEST / ACTIVE / INACTIVE。OFFLINE 是 ACTIVE 的子态，不独立。

---

## 4. 用户路径

### 4.1 主路径：未登录 → 激活（happy path）

```
触点 → [统一登录卡] → OAuth → [合并进度卡 3✓] → ACTIVE 设置页
触点 = {首页 chip, 对话页地球图标/胶囊, 侧栏账号块, 设置 toggle}
全程 ≤ 6 步交互（含 OAuth 环节）
```

### 4.2 已登录·关 → 激活

- 设置页 toggle 直接打开 → 短暂建链过场 → 激活
- 不再弹登录卡
- 对话页地球图标点击 → 同样直接开

### 4.3 异常

- **离线**：toggle 保持 ON，状态条变红 ⚠️ + 重连按钮，专属地址保留
- **OAuth 失败/取消**：进度卡停在第一步，重试/取消
- **退账号**：强确认 → 关链 + 清地址 + 清设备 → GUEST

## 5. 入口表现规则

### 5.1 首页底部 earth chip

| 状态     | 显示                                          | 行为                                 |
| -------- | --------------------------------------------- | ------------------------------------ |
| GUEST    | "🌐 扫码即可远程 · **登录解锁** →" + 渐变吸引 | 点击弹登录卡（intent=remote）        |
| ACTIVE   | "🌐 远程连接 · 已开启" 绿色点                 | 点击弹气泡（地址 / 设备 / 进入设置） |
| INACTIVE | **不显示**                                    | —                                    |
| OFFLINE  | "🌐 远程已开 · 中继断开" 黄色 ⚠️              | 点击弹气泡（重连）                   |

### 5.2 对话页右上角 🌐 地球图标（v2.1 新增，四态常驻）

| 状态     | 颜色          | hover tooltip            |
| -------- | ------------- | ------------------------ |
| GUEST    | 蓝（primary） | "登录解锁手机访问"       |
| ACTIVE   | 绿 + 小绿点   | "远程已开 · iPhone 在线" |
| INACTIVE | 灰            | "开启远程可手机继续看"   |
| OFFLINE  | 黄 ⚠️         | "中继断开 · 点此重连"    |

点击 → 同首页 chip 弹气泡逻辑。位置：靠近 Claude Code chip。

### 5.3 对话页长任务引流胶囊

- **触发条件**：（GUEST 或 INACTIVE）且当前会话有任务跑 ≥ 30s
- **关闭按钮 ×**：点 × → snooze 24h，期间任何会话都不再弹（仍保留右上角地球图标作为入口）
- **同会话内**：每会话最多弹一次
- **ACTIVE / OFFLINE**：不显示

文案：

- GUEST：「📱 任务还在跑？登录后可在手机继续看进度 [登录]」
- INACTIVE：「📱 任务还在跑？开启远程可在手机继续看 [开启远程]」

### 5.4 侧栏底部账号块

| 状态     | 显示                             |
| -------- | -------------------------------- |
| GUEST    | 灰头像「未登录 · 点此登录」      |
| ACTIVE   | 头像 + 邮箱 + 绿点「远程已连接」 |
| INACTIVE | 头像 + 邮箱 + 灰「远程已关闭」   |
| OFFLINE  | 头像 + 邮箱 + 黄 ⚠️「中继断开」  |

点击 → 弹账号面板（含「前往设置 / 退出账号」）。

### 5.5 设置页

- toggle 永远是当前状态的真实开关
- GUEST 点 toggle → 弹登录卡（intent=remote）
- INACTIVE 点 toggle → 短暂过场后激活，不弹登录
- ACTIVE 点 toggle → 直接关闭（不弹确认，因为重新开很轻）
- 已激活态展示：状态条 + 专属地址 + 二维码（远程模式）+ 设备列表

## 6. 登录卡（统一组件）

**单卡多 intent，登录 + 建链合一**

```
┌────────────────────────────────────┐
│ 🔐 登录 Aion 账号                  │
│  · 用于身份识别、Token 计费        │
│  · 仅读取邮箱，不访问邮件或文件     │
│                                    │
│  [G] 使用 Google 账号继续          │
│  ↓ 仅当 intent=remote 时显示       │
│  「登录后将自动开启 WebUI 远程访问，│
│   可在设置中关闭」                 │
│                                    │
│  取消                              │
└────────────────────────────────────┘
        │
        ▼ OAuth 成功后，同卡推进（不切组件）
┌────────────────────────────────────┐
│ ✓ 账号登录成功 · linveer15@gmail.com│
│ ✓ 已分配专属地址 abc12.aion.app    │
│ ⟳ 与中继服务器建立连接中… [进度条] │← intent=account 时跳过此步
└────────────────────────────────────┘
        │
        ▼ ≤ 5s 后自动收起
   设置页常驻"已激活"状态
```

**关键差异 vs v2.0**：

- ❌ 删除"步骤 1/3 登录"+"步骤 2/3 建链"两张分卡
- ✅ 同一卡内推进 3 个 ✓
- ✅ intent=account 时只走前两个 ✓，不建链

## 8. 频次控制（snooze）

- 用户点 × 关闭对话页胶囊 → `localStorage.setItem('aion-promo-snooze', Date.now() + 24h)`
- 检查时机：每次进入对话页 + 任务跑 ≥ 30s 时
- 已 snooze 期间：胶囊不弹，但右上角 🌐 地球图标**仍在**（用户随时可手动入口）

---

## 9. 验收 DoD

- [ ] 首次未登录用户从看到任意触点 → 完成登录 → 看到二维码可扫，**全程 ≤ 6 步交互**
- [ ] GUEST 路径：触点 → 登录卡 → OAuth → 进度卡 3✓ → 已激活
- [ ] INACTIVE 路径：toggle/地球图标 → 进度卡（仅建链）→ 已激活，**无登录卡**
- [ ] inactive 在首页 chip 不曝光，但侧栏/对话右上仍可见
- [ ] 离线 toggle 保持 ON
- [ ] × 胶囊后 24h 不再弹
- [ ] 二维码在 active/offline 都是远程模式（abc12.aion.app）
