# 3D Morph PPT 助手：1.0 vs 3.0 对比与 SKILL 解读

## 一、版本对比

| 维度                        | **1.0** (旧版)                   | **3.0** (新版)                                                  |
| --------------------------- | -------------------------------- | --------------------------------------------------------------- |
| **继承 morph-ppt 设计体系** | ❌ 独立运行，没有设计规范        | ✅ 继承全套（配色/字体/版式/ghost/验证）                        |
| **SKILL 行数**              | 106 行                           | ~240 行                                                         |
| **模型摆放规则**            | 无（只有 add 命令语法）          | ✅ 60/40 原则 + 4 种布局模式                                    |
| **模型与内容关系**          | 无                               | ✅ 模型区/内容区分离，不重叠                                    |
| **模型与画幅适配**          | 无                               | ✅ 按页面类型给模型尺寸范围                                     |
| **镜头语言**                | "delta >= 20" 一句话，只有左右转 | ✅ 7 种镜头类型（全景/特写/鸟瞰/仰拍/侧面/过肩/美拍），推拉节奏 |
| **版式多样性**              | 无                               | ✅ 相邻页必须切换布局                                           |
| **禁止 clone 3D 模型**      | 未提及                           | ✅ 明确禁止                                                     |
| **禁止 raw-set 3D 参数**    | 允许使用                         | ✅ 明确禁止                                                     |
| **Morph 命名/ghost**        | 未提及                           | ✅ 继承（`!!scene-`/`!!actor-`/`#sN-`）                         |
| **验证流程**                | 仅 validate                      | ✅ final-check + 截图 + 3D 专属检查                             |
| **提示词**                  | 技术导向（"修复视角"）           | ✅ 创作导向（产品展示、故事叙事）                               |

**一句话总结：** 1.0 只管"怎么把模型塞进 PPT"，3.0 管"怎么用电影镜头语言做出好看的 3D PPT"。

## 二、3.0 SKILL 完整解读

以下是 3.0 SKILL.md 的逐节翻译和解释。

---

### 2.1 定位

> 本技能**扩展** morph-ppt。morph-ppt 的所有规则（命名、ghost、设计、验证）全部适用。
> 本文件只写 **3D 相关的增量规则**。

**含义：** 3.0 不是独立的助手，而是 morph-ppt 1.0 的"3D 插件"。PPT 的设计、配色、字体、Morph 动画规范全用 1.0 的，3D 只增加模型相关的规则。

---

### 2.2 模型兼容检查（生成前必做）

1. 只支持 `.glb` 格式。用户给 `.fbx`/`.obj`/`.blend`/`.gltf` 要先转换
2. 用户没有模型时推荐下载来源：Sketchfab、Poly Pizza、Khronos 样例、Poly Haven
3. 所有文件（模型、PPT、脚本）必须在同一个工作目录

**含义：** 在开始做之前先确认模型能用。避免生成到一半发现格式不对。

---

### 2.3 模型插入规则

#### 每页单独 add，绝对不能 clone

> `morph_clone_slide` 会把模型复制为冻结的 XML 副本。被复制的模型无法 Morph 动画。
> 每页必须独立调用 `add --type 3dmodel`，使用**相同的 name**。

**含义：** Morph 转场需要 PowerPoint 在两页之间"匹配"同名的 3D 对象。clone 出来的是死的 XML，PowerPoint 不认为它是同一个对象，所以不会产生过渡动画。

#### 可以控制的属性

| 属性              | 作用               | 说明                           |
| ----------------- | ------------------ | ------------------------------ |
| `x`, `y`          | 模型在页面上的位置 | 标准坐标                       |
| `width`, `height` | 模型框的大小       | 模型在这个框里渲染             |
| `name`            | 形状名称           | 所有页必须一样，Morph 才会匹配 |
| `roty`            | Y 轴旋转（度数）   | 主要的叙事控制轴               |
| `rotx`            | X 轴俯仰（度数）   | 轻微使用，5-20° 做微妙倾斜     |
| `rotz`            | Z 轴翻滚（度数）   | 很少需要                       |

#### 绝对不能手动设置的

- `meterPerModelUnit` — officecli 从 GLB 自动算
- `preTrans` — officecli 自动居中
- `camera` 深度/位置 — officecli 自动适配模型
- 不要对任何 3D 变换参数使用 `raw-set`

**含义：** 以前 1.0/2.0 的核心问题就是手动覆盖了 officecli 自动计算的参数，导致模型消失或视角错乱。3.0 明确禁止这些操作。

---

### 2.4 模型与内容的布局（核心设计规则）

#### 60/40 原则

> 模型是**视觉主角**，不是缩略图装饰。
> 模型区：占页面宽度的 50-65%（通常 16-22cm 宽）
> 内容区：占页面宽度的 35-50%
> 模型和内容**不能重叠**

**含义：** 以前 2.0 生成的 PPT 里，模型被放在角落很小的位置，跟内容挤在一起。3.0 要求模型占主导地位。

#### 4 种布局模式

| 模式                      | 说明                                          | 适用场景               |
| ------------------------- | --------------------------------------------- | ---------------------- |
| **A：模型右，内容左**     | 最常见。内容在左边 2-14cm，模型在右边 15-32cm | 常规内容页             |
| **B：模型左，内容右**     | 跟 A 对调，制造节奏变化                       | 交替使用，避免单调     |
| **C：模型居中，文字叠加** | 模型占满中央，标题在上方或下方                | 标题页、封面页、结束页 |
| **D：模型小，内容为主**   | 模型缩小到 8-10cm 放角落，内容填满            | 数据页、细节页         |

#### 版式节奏

> 相邻两页不能用同一种布局。必须交替变化：
>
> 第 1 页（标题）：模式 C — 模型居中
> 第 2 页（内容）：模式 A — 模型右
> 第 3 页（内容）：模式 B — 模型左
> 第 4 页（数据）：模式 D — 模型角落
> 第 5 页（内容）：模式 A — 模型右（不同位置）
> 第 6 页（结尾）：模式 C — 模型居中

**含义：** 以前生成的 PPT 里，每页模型都在同一个位置，看起来死板。3.0 要求有视觉节奏。

#### 模型尺寸指南

| 页面类型     | 模型宽度 | 模型高度 | 目的           |
| ------------ | -------- | -------- | -------------- |
| 标题/封面    | 18-22cm  | 15-18cm  | 气势磅礴       |
| 内容（分屏） | 14-18cm  | 13-16cm  | 与文字平衡     |
| 细节/数据    | 8-12cm   | 8-12cm   | 点缀，不抢焦点 |
| 结尾         | 16-20cm  | 14-17cm  | 情感锚点       |

**含义：** 不同类型的页面，模型应该有不同的"存在感"。标题页要大气，数据页要克制。

---

### 2.5 镜头语言（电影化编排）

> **把每一页当作一个镜头，不是一个旋转角度。**

3.0 有三个镜头工具配合使用：

- **roty**：左右环绕（转盘）
- **rotx**：上下俯仰（摇臂）—— 范围 -25° 到 +40°
- **width/height**：推近拉远（特写 vs 全景）

#### 7 种镜头类型（每份 PPT 至少用 3 种）

| 镜头类型                      | 模型大小                | rotx       | 效果                   | 适用场景               |
| ----------------------------- | ----------------------- | ---------- | ---------------------- | ---------------------- |
| **全景（Establishing）**      | 大（18-22cm）           | 0-5        | 完整模型，平视         | 标题页、开场、结尾     |
| **3/4 美拍（Beauty）**        | 大（16-20cm）           | 5-10       | 经典产品角度           | 主视觉页、第一印象     |
| **特写（Close-up）**          | 超大（22-28cm），裁切   | 0-10       | 模型填满画面，边缘被裁 | 功能亮点、材质、细节页 |
| **鸟瞰（Bird's eye）**        | 中（14-18cm）           | 25-40      | 从上往下俯视           | 结构、内部布局、总览   |
| **仰拍（Low angle）**         | 中大（16-20cm）         | -15 到 -25 | 从下往上仰视，有气势   | 力量感、权威、高潮     |
| **侧面（Side profile）**      | 中（14-16cm）           | 0          | 剪影轮廓               | 人体工学、外形对比     |
| **过肩（Over-the-shoulder）** | 中小（10-14cm），偏一侧 | 10-15      | 模型做背景，内容为主   | 数据页、详细文字页     |

**含义：** 以前只有 roty 左右转，像模型站在转盘上。现在加入俯拍（从上往下看结构）、仰拍（从下往上看气势）、特写（推近看细节），让镜头真正"动"起来。

#### 特写技巧

把模型做得**比画面更大**，让边缘被裁切，制造"推近"的感觉：

```
模型 width=26cm, height=22cm, x=4cm, y=-3cm
→ 模型超出画面边界，被自然裁切
→ 效果类似电影里的特写镜头
```

**含义：** 以前每页模型都是完整显示的，看起来都差不多。特写让某些页面"推进去"，观众感受到距离变化。

#### 内容驱动镜头（核心原则）

> 镜头类型必须**服务于当前页的内容**。

决策流程：

1. 先看这一页讲什么内容
2. 问："如果这是电影，摄影师会用什么镜头？"
3. 选镜头类型 → 设 roty + rotx + 模型大小
4. 确认相邻页有足够的视觉差异

举例：

- "正面设计" → **特写**，`roty=0, rotx=5`，模型 24cm 宽（被裁切）
- "侧面轮廓" → **侧面**，`roty=90, rotx=0`，模型 15cm
- "内部结构" → **鸟瞰**，`roty=30, rotx=35`，模型 16cm
- "气势/力量感" → **仰拍**，`roty=20, rotx=-20`，模型 20cm
- "整体概览" → **3/4 美拍**，`roty=40, rotx=8`，模型 18cm
- "数据与参数" → **过肩**，`roty=60, rotx=10`，模型 12cm 在角落

#### 推拉节奏（Zoom Breathing）

> 模型大小必须有"呼吸感"，不能连续 3 页同尺寸。

```
第 1 页：大    （全景）
第 2 页：超大  （特写）    ← 推进
第 3 页：中    （拉回）    ← 拉出
第 4 页：中    （不同角度）
第 5 页：大    （高潮）    ← 推进
第 6 页：小    （数据）    ← 拉出
第 7 页：大    （结尾）    ← 推进
```

**含义：** Morph 转场会自动在两页之间做模型大小的动画过渡。如果每页模型都一样大，这个动画效果就浪费了。大→小→大的"呼吸"让每次翻页都有推近/拉远的镜头运动感。

**含义：** 以前的旋转规划是"每页 +50°, +50°, +50°"这种机械方式。3.0 要求 AI 先理解内容，再选视角。这是 3.0 最重要的改进之一。

#### 内容驱动旋转示例

| 页  | 内容           | 为什么选这个角度   | roty | rotx |
| --- | -------------- | ------------------ | ---- | ---- |
| 1   | 标题——第一印象 | 最好看的 3/4 视角  | 30   | 5    |
| 2   | 正面特征       | 展示正脸           | 0    | 0    |
| 3   | 侧面轮廓       | 展示剪影           | 80   | 0    |
| 4   | 内部结构       | 稍微俯视，转过来   | 130  | 15   |
| 5   | 背部细节       | 背面               | 180  | 0    |
| 6   | 总结/全貌      | 另一侧的 3/4 视角  | 250  | 10   |
| 7   | 结尾           | 回到接近开场的角度 | 330  | 5    |

#### 旋转技术规则

1. 相邻页 roty 差值 >= 30°（太小看起来像抖动）
2. 相邻页 roty 差值 <= 90°（太大会让人眩晕）
3. 整体方向一致（不要来回跳：0→90→30→120）
4. 总弧度 180-360°（半圈或一圈，让观众看到模型全貌）

### 2.6 与 morph-ppt 的工作流整合

#### Phase 2（规划）新增内容

在 `brief.md` 里，除了标准的 Morph Pair Scene 表，还要加一个 **模型编排表**：

| 页  | 布局模式    | 模型 x,y,w,h | roty | rotx | 内容位置    |
| --- | ----------- | ------------ | ---- | ---- | ----------- |
| 1   | C（居中）   | 7,0.5,20,17  | 30   | 5    | 底部标题    |
| 2   | A（模型右） | 16,1,16,16   | 0    | 0    | 左侧 2-14cm |
| ... | ...         | ...          | ...  | ...  | ...         |

**含义：** 在动手写代码之前，先把每一页的模型位置、角度、内容位置全部规划好。避免写到一半发现布局不协调。

#### Phase 3（构建）差异

因为 3D 模型不能 clone，构建顺序不同于标准 morph-ppt：

1. 先创建所有 slide（背景 + morph transition）
2. 在第 1 页加场景演员（`!!scene-*`），然后 clone 到后续页（场景演员可以 clone）
3. 每页**独立** add 3D 模型（同名、不同 roty/位置）
4. 每页加内容文字，ghost 上一页的内容

#### Phase 4（验证）新增检查

除了标准的 morph 验证，额外检查：

- 每页有且只有一个 `model3d` 元素
- 所有模型的 `name` 相同
- 模型框在画面范围内
- 模型和内容文字没有重叠
